v.138: Walking Paths overlay — HAL accessible-route lines per deck

* New togglable "Walking paths" pill above the deck picker.
  Pressing it sets body.show-routes which fades route-NN.png
  layers in over each deck plan. Preference persists per session.
* 11 new route overlay PNGs (route-01..route-11.png), one per
  passenger deck. Generated from the HAL Zuiderdam "Accessible
  Routes — effective Dec 5, 2025" PDF: each panel cropped, rotated
  90 CCW (PDF stern-left / bow-right -> portrait bow-up), scaled
  per-deck to anchor bow + stern to the existing deck-NN.png
  content bbox, then masked to keep only the orange route color
  (re-tinted to brass) on a transparent background. Short decks
  (8-11) use their own anchor pair so the routes sit on the
  actual deck length rather than getting stretched.
* New inline CSS: position-absolute .fp-route-img layer per
  .fp-img-stage. opacity 0 by default, .25s fade on toggle,
  z-index 2 (below hotspots so taps still register on venues).
* Hotspots forced to position: absolute + z-index: 3 inside the
  stage so they stay clickable above the overlay.
* Asset versions bumped to ?v=138 across the page; app.js
  SITE_VERSION = 'v.138'.
