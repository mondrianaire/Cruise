v.142: restore missing mouseleave handler in hover-sync

Symptom: on desktop, hovering a "Notable on this deck" row (or its
matching deck-plan hotspot) added the tan .is-hover highlight but
the row stayed highlighted after the cursor moved away. Multiple
rows ended up "selected" simultaneously because nothing was
clearing the class.

Root cause: the pair() function in the inline hover-sync IIFE was
missing its mouseleave registration:

    el.addEventListener('mouseenter', on);
    // <-- el.addEventListener('mouseleave', off); was here
    el.addEventListener('focusin', on);
    el.addEventListener('focusout', off);

The line got dropped during the v.139 tail-restoration splice. The
"on" handler ran on hover, added .is-hover, then nothing removed
it (focusout only fires if the row was focused, which hovering
alone doesn't do).

Caught by live Chrome inspection: dispatched mouseenter +
mouseleave on a dv-item, instrumented classList.remove to log
calls — removeLog was empty after mouseleave, proving no handler
was running. Source-grep then showed the missing line.

Fix: add the mouseleave registration back. Asset versions bumped
to ?v=142. SITE_VERSION = 'v.142'.
