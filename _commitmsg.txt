v.136: kill the deck-switch blue flash

Root cause: every deck PNG was tagged loading="lazy". When the user
switched from Deck 11 to Deck 5, Deck 5's image only started
downloading at that moment - the panel briefly showed an empty
container which iOS renders with its default pale-blue placeholder.
Combined with a hard display:none -> display:block swap on the
.fp-panel, this read as a jarring blue flash.

Fixes:

1. Removed loading="lazy" on all 11 deck images. Total weight is
   modest (each is ~50 KB) so eager-loading on view enter is cheap
   and switches are now instant.

2. .fp-img-stage background painted in the deck-paper cream
   (#fbf6e4) so even if the image takes a frame to paint, the
   placeholder is cream - matches the deck plan - not iOS blue.

3. Same cream background on the .fp-deck-img element itself in
   case it ever renders before the bitmap is decoded.

4. Mobile select and deck-button tap highlight set to transparent
   so the iOS blue tap state can't flash either.

5. Slightly longer panel reveal animation (0.18s instead of 0.25s)
   for snappier crossfades.

Cache buster -> ?v=136.
