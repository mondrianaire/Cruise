v.121: defeat iOS Safari cache - inline critical CSS + ?v= cache busters

Frames from screen recording showed the v.120 deployment landing with
1) Night Owl button STILL visible in deck plans, and 2) orientation
gate still not appearing. Root cause was iOS Safari serving cached
styles.css that didn't contain the v.118+ rules. The HTML and JS were
fresh (version pill showed v.120 from fresh app.js) but the actual
visual rules were missing.

Fix:
- Critical CSS rules now inlined in a <style id="ship-critical-v121">
  block inside ship.html, immediately after the <link rel="stylesheet">.
  Includes the full .rotate-overlay structure + .ship-no-toggle[hidden]
  + body.view-floor .ship-no-toggle hides. These rules ship with the
  HTML and can't be cached separately from it. Whatever version of
  styles.css the CDN serves, these rules WILL apply.
- Cache buster query string added: styles.css?v=121, app.js?v=121.
  iOS Safari treats different URLs as separate cache entries, so the
  next load will pull fresh copies of both.
- Phone rotation glyph in the orientation gate simplified - was an
  animated SVG that depended on @keyframes in styles.css; now an
  inline emoji 📱↻ inside a brass-bordered square, all styled by the
  inline critical CSS. No external dependency.
- z-index bumped to 2147483647 (max int32) so nothing can layer
  over it.
