v.131: deep-link URL support + venue catalog for cross-referencing events

Preparing the ingestion pipeline for Navigator-app daily program events.

ship.html now accepts URL deep-link params:
  ship.html?view=floor&deck=N&highlight=venueId
  ship.html?view=elev

Anyone linking from anywhere on the trip hub to a specific ship venue
lands on the deck-plans view, the requested deck panel, and the venue
flashes in the venue list. Mobile portrait gets the existing
orientation gate so the deck plan view loads in portrait by default.

Implemented as applyDeepLink() at the end of the IIFE - sets the
shipViewOverride flag so the auto-orient logic doesn't fight, then
sequences setTab('floor') -> setDeck(N) -> highlightVenue(deck, id).

New shared venues.js module with 70 venue-key entries mapping
Navigator-style location strings ("Ocean Bar, Deck 3", "Spa & Salon,
Deck 9", etc.) to {deck, venueId} pairs. Includes fuzzy fallback for
"the X" prefix stripping. window.shipVenueLink(loc) returns
{ url, deck, id, label } or null. Both ship.html and journey.html now
load venues.js.

Catalogues venues that aren't yet on the cross-section so the linker
gracefully falls back to the closest neighbour:
  Explorer's Lounge   -> lincolnctr (same Music Walk room)
  Rolling Stone       -> billboard  (same Music Walk row)
  Library             -> shops      (Deck 3 cluster)
  Game Room           -> loft       (same teen/lounge area)
  Art Studio          -> explorations (adjacent on Deck 10)
  Shuffleboard Court  -> crowsnest  (aft of obs deck)
  Hudson Room         -> meetingrooms
