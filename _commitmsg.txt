v.141: route img scale bug + drawer carryover on deck switch

Two regressions from v.139/140 caught with live Chrome inspection:

ROUTE OVERLAY SCALE: .fp-img-stage had `padding:6px` in floor view
(min-width:901px). The deck img (width:100%/height:100%) sizes
to the stage's CONTENT BOX (118-12 = 106px wide) and then
object-fit:contain letterboxes inside. The route img however was
position:absolute / inset:0 which references the PADDING BOX (118px
wide), so it spilled 12px wider every direction and rendered at a
different scale. Looked like "the overlay is from a different
floor" because the routes didn't line up with the deck features.
Fix: removed the 6px stage padding in floor view. Both layers now
size against the same box and letterbox to the same rendered
rectangle.

DRAWER CARRYOVER: switching decks left any open venue drawer in
the previous panel still flagged is-open. Hidden panel kept the
state, so returning to that deck showed the old drawer expanded.
Fix: setDeck() now calls window.closeAllVenueDrawers() (exposed
from events-drawer.js) on every deck switch.

Verified live in Chrome: route img dimensions now match deck img
dimensions (both 104x749 in the test viewport); drawer toggle
state cleared on deck switch.

Asset versions bumped to ?v=141. SITE_VERSION = 'v.141'.
