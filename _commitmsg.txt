v.118: iOS-safe overlay reveal + belt-and-suspenders Night Owl hide

Two reported bugs:

1. Night Owl toggle was still visible in deck plans view.
   Root cause: `.ship-no-toggle{display:inline-flex}` (base style) has the
   same source-order weight as the browser-default `[hidden]{display:none}`
   and since it appears later in the cascade, it wins -- so setting
   noBtn.hidden = true had no visual effect. The
   `body.view-floor .ship-no-toggle{display:none!important}` rule should
   have caught it but evidently wasn't matching reliably in every layout
   pass (possible Safari race with the v.110 grid restructure).
   Added a class+attribute selector
   `.ship-no-toggle[hidden]{display:none!important}` that always wins
   regardless of body class.

2. iOS Safari couldn't render the rotate overlay at all.
   The overlay used `opacity:0` + `transition:opacity` and depended on
   `requestAnimationFrame` to flip the class. On iOS Safari the
   transition would occasionally never fire and the overlay sat at
   opacity:0 forever -- the user saw nothing.
   Rebuilt the reveal:
     - CSS `display:none` by default; `.visible` flips to `display:flex`
       with a CSS animation (not transition).
     - `inset:0` replaced by explicit top/right/bottom/left for iOS < 14.5.
     - Card pop-in is now an animation too.
     - JS no longer toggles the `hidden` attribute. Show = add `.visible`;
       hide = remove `.visible`.
   Applied to both the initial portrait orientation gate and the
   landscape deck-pick prompt.
