ingest: verify.py + retry_ocr.py + venue catalog cross-check pipeline

Added /ingest/ folder with reusable verification system for the daily
program OCR pipeline.

verify.py - five cheap verifiers per event:
  1. field_completeness  - placeholder/null detection in time/name/venue/deck
  2. duration_canonical  - duration matches HAL cadence (15/30/45 min, 1-4 hr,
                           13-17 hr all-day) or is an acceptable absence
  3. venue_in_catalog    - venue + deck exists in shared SHIP_VENUE_CATALOG
  4. title_quality       - rejects OCR artifacts (e.g., "wat3rcoloring") and
                           uncertainty markers ("[?]", "(?)")
  5. time_order          - events sorted, no backwards jumps

Each verifier contributes to a 0-5 score. >=4 HIGH, ==3 MEDIUM, <=2 LOW.

retry_ocr.py - targeted re-OCR for MEDIUM/LOW events:
  - Scan frames at sparse interval for the event's title fingerprint
  - Rank candidates by Laplacian-stdev sharpness
  - Re-OCR the sharpest frame, return field corrections

Tested on a Day 2 fixture with intentional bad rows. Caught:
  - "37 minutes" Mahjong  -> non-canonical duration + wrong deck + back-in-time
  - "wat3rcoloring"       -> suspicious digit-in-word
  - duplicate Vivace Duo  -> time-order violation
All HIGH-marked events confirmed legitimate.

This module is the standard pre-commit step for every day's extracted
events going forward. Pipeline:
  ffmpeg extract -> first-pass OCR -> verify.py -> retry_ocr for any
  MEDIUM/LOW -> final JSON with venue deep-links via venues.js.
